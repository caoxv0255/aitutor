- **分层架构**：采用典型的 Web 服务分层，`main.py` 作为 FastAPI 入口处理 HTTP 请求，`indexer.py` 负责后台索引调度，`db.py` 封装 PostgreSQL 数据访问，`config.py` 集中管理环境变量与索引配置。
- **外部集成**：通过 `subprocess` 调用官方 `graphrag` CLI 执行底层的索引构建（`graphrag index`）和语义查询（`graphrag query`），并动态生成 `settings.yaml` 适配不同索引场景。
- **数据持久化**：使用 `psycopg2` 连接 PostgreSQL，维护文档状态（`graphrag_documents`）、索引任务进度（`graphrag_index_jobs`）及查询日志（`graphrag_query_logs`），实现断点续跑与任务追踪。
- **异步调度**：索引模块利用 `asyncio` 结合 `tenacity` 实现带指数退避的重试机制，并通过自定义 `RateLimiter` 控制对 LLM API 的调用频率，防止触发限流。
